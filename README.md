## Overview

A production-architected in-car voice assistant that processes speech, vision, and natural language across a full agentic pipeline. Speak a command → GPT-4o-mini classifies intent → safety gate screens it → tools execute → PyQt5 dashboard updates in real time → Cartesia speaks the response.

**Three modalities, one pipeline:**
- **Voice** — Deepgram STT + Cartesia TTS via Pipecat, sub-1.5s round-trip
- **Vision** — GPT-4o vision answers questions about dashcam/cabin images
- **Action** — LangGraph agent routes to climate, navigation, or vision tools

### Safety Gate Ablation (Rule-based vs LLM)
Ran both classifiers on 15 test cases to compare accuracy:

| Classifier | Agreement | False-allows | False-refuses |
|---|---|---|---|
| Rule-based | 86.7% | 0 | 0 |
| GPT-4o-mini | 86.7% | 0 | 2 |

**Finding:** Rule-based gate is more reliable for this use case — zero errors vs. 2 LLM false-refuses. The LLM over-refuses edge cases ("take a photo while parked"). Deterministic rules win on safety-critical classification.

## Architecture Decisions

**GPT-4o-mini for intent classification**
Simple 4-class classification task. Fast (< 1s), cheap ($0.00015/call), and accurate. In production, this would be the first component to replace with an on-device model (Phi-3-mini, Qwen2.5-1.5B) to eliminate cloud dependency and latency.

**GPT-4o vision instead of Qwen2-VL locally**
Qwen2-VL-7B requires ~16GB VRAM. For portfolio demonstration, GPT-4o vision gives equivalent quality without GPU infrastructure. The `VisionAssistant` class is architected to swap — implement `_lazy_load()` in `agent/vision.py` for the local model path.

**Deepgram + Cartesia**
Deepgram Nova-2 has best-in-class WER for conversational speech. Cartesia produces natural TTS at low latency. Both have first-class Pipecat integrations, keeping the voice pipeline clean.

**Rule-based safety gate**
Deterministic, auditable, and fast — no API call on the critical path. The ablation study confirms this was the right call: 0 errors vs. 2 false-refuses for GPT-4o-mini on 15 test cases. An LLM safety classifier introduces hallucination risk on safety-critical decisions.

**LangGraph over custom agent loop**
The classify → safety → route → respond flow maps naturally to a state machine. Each node is independently testable. The main tradeoff is an extra dependency; the benefit is built-in visualization and easy extension.

## Limitations & Future Work

| Limitation | Impact | Fix |
|---|---|---|
| Cloud APIs only | No offline capability | Replace with on-device models (Phi-3, Qwen2.5) |
| Driving state is a manual flag | Not sensing real vehicle state | Read CAN bus: speed > 0, gear ≠ Park |
| Hand-built safety rule list | Cannot generalize to novel threats | Train a safety classifier on automotive datasets |
| Static sample image | Vision always sees same frame | Capture live camera frames per query |
| No conversational memory | Each command is stateless | Add LangGraph memory module |
| Single-shot commands only | "Make it the train station instead" breaks | Add multi-turn context buffer |
